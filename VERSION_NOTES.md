# 版本更新说明 - 逻辑对齐与社交推演优化 (v0.5.0)

本版本专注于修复核心规则逻辑、提升 AI 认知一致性以及增强说书人（Storyteller）的全局控制体验。

## 核心更新特性

### 1. 认知一致性重构 (Identity Alignment)
- **Agent 强制同步**：修复了在准备阶段（Setup）角色洗牌后，人类玩家（h1）和预存在 Agent 身份滞后的 Bug。现在所有角色在游戏正式开始前会强制同步最新的 `PlayerState`。
- **酒鬼 (Drunken) 逻辑完善**：现在酒鬼能够正确感知自己的假身份，并在思考过程中以该假身份进行推理，而后台引擎则按酒鬼规则（能力失效/假信息）执行。

### 2. 说书人魔典与全知视野 (Storyteller Dashboard)
- **上帝视角集成**：说书人 (`h1`) 现在可以实时接收所有 `PRIVATE` 和 `STORYTELLER_ONLY` 事件。即使是 AI 的私密夜间动作和系统结算结果，也会实时呈现在说书人的日志中。
- **魔典可视化**：集成式的 Grimoire 现在能清晰展示中毒（Poisoned）、醉酒（Drunk）、死亡（Dead）以及受保护（Protected）等状态徽章。

### 3. 规则引擎审计 (Rule Engine Audit)
- **行动顺序校准**：按照官方《暗流涌动》剧本对 22 个角色的 `night_order` 进行了精确校准（例如：投毒者先于预言家动作）。
- **技能逻辑补全**：
    - **预言家 (Fortune Teller)**：实现了“红鲱鱼”宿敌机制，保证每局有一位好人被视为恶魔。
    - **洗衣妇/图书馆员/调查员**：完善了首夜信息分发模板。
    - **小恶魔 (Imp)**：修复了自杀传位的稳定性。

### 4. 提名与处决流程优化 (Nomination Flow)
- **序列化互动**：重构了白天提名流程，支持“提名 -> 辩解发言 -> 实时投票”的顺序逻辑。
- **UI 交互增强**：（计划中/已部分实现）增加了更清晰的消息类型区分，使社交推演过程更具沉浸感。

### 5. 开发者工具 (Dev Ops)
- **MockBackend**：新增模拟 LLM 后端，支持在无 OpenAI API Key 的环境下进行全流程自动化规则审计。
- **模拟审计脚本**：升级后的 `simulate_game.py` 支持 8-15 人对局的逻辑压力测试。

---
*本版本致力于打造最严谨的《血染钟楼》AI 推演引擎，确保每一项技能发动都符合官方规则书。*
