# Alpha 0.2 专项计划：说书人平衡裁量模拟数据与评估

## 背景

`W2-A` 的核心不是单纯把夜晚信息链写通，而是让说书人真正具备“可信的自由裁量能力”。

在《血染钟楼》中，说书人的最高目标不是机械执行规则，而是在规则允许的边界内主持一局尽兴、紧张、悬念持续到后期的游戏。因此，说书人智能优化不能只靠少量单元测试，也不能只靠人工对几局日志做直觉判断。

必须建立一套可复用的模拟对局数据与评估体系，让我们可以持续回答这些问题：

- 这次裁量是否仍在规则允许边界内？
- 这次假信息是否“合理而不过分”？
- 这次平衡调整是否避免了游戏过早失去悬念？
- 说书人是否在“保证公平”和“维持戏剧性”之间取得了合适平衡？

---

## 总体目标

建立一套面向说书人裁量能力的离线评估体系，覆盖：

1. 完整对局状态快照采样
2. 说书人裁量决策回放
3. 规则边界校验
4. 平衡性与戏剧性评估
5. 自动化 acceptance gate

---

## 一、为什么需要模拟对局数据

当前已有的测试大多解决的是：

- 规则是否正确
- 事件顺序是否正确
- API 输出是否稳定

但说书人的自由裁量还涉及另一层问题：

- 当多种合法假信息都可选时，该选哪一种？
- 当善方或恶方明显劣势时，如何在规则边界内给出更“尽兴”的裁量？
- 当游戏可能在前 2 天就失去悬念时，是否存在更好的裁量路径？

这些问题不能只用“单条断言”评估，而更适合用“一整局上下文”去评估。

因此，我们需要生成一批**包含整局上下文的模拟对局数据**，作为说书人裁量测试和后续智能增强的基础资产。

---

## 二、数据集的目标形态

每一条样本不应只是单个 night info，而应是一个“裁量时刻包”。

### 1. 基础结构

每个样本至少包含：

- `game_id`
- `script_id`
- `seed`
- `round_number`
- `phase`
- `seat_order`
- `players_truth`
- `players_public_state`
- `players_private_delivery_history`
- `event_log_so_far`
- `current_effects`
- `storyteller_context`
- `candidate_adjudications`
- `chosen_adjudication`

### 2. `players_truth`

仅供说书人评估使用，包含：

- 真实角色
- 当前阵营
- 是否存活
- 中毒 / 醉酒 / 保护 / 失能状态
- 转职 / 接管历史

### 3. `players_public_state`

包含玩家公开可见局势：

- 历史发言摘要
- 历史提名与投票记录
- 死亡与处决记录
- 公开宣称身份

### 4. `players_private_delivery_history`

包含已经由说书人发给玩家的私密信息历史：

- 首夜信息
- 夜晚信息
- 死亡触发信息
- 间谍魔典视图

### 5. `storyteller_context`

用于让说书人理解“局势温度”，建议包含：

- 当前善恶人数
- 当前是否有明显信息位崩塌
- 当前是否有人类玩家参与
- 当前局势是否已经接近锁死
- 当前回合结束后潜在胜负分支

---

## 三、样本类型设计

为了训练和评估说书人，我们至少要有 4 类样本。

### A. 固定信息裁量样本

例如：

- 洗衣妇 / 图书馆员 / 调查员
- 厨师 / 共情者 / 送葬者
- 占卜师 / 守鸦人 / 间谍魔典

目标是评估：

- 真实信息是否正确
- 假信息是否合理
- 假信息是否能解释

### B. 状态扰动样本

例如：

- 酒鬼
- 中毒
- 隐士 / 间谍误判
- 红鲱鱼

目标是评估：

- 扰动是否落在规则边界内
- 扰动是否没有“过度破坏公平”

### C. 平衡干预样本

例如：

- 善方过强
- 恶方过强
- 前两天可能直接锁死
- 核心信息位被连续压制

目标是评估：

- 说书人是否会倾向于让游戏悬念延续到后期
- 是否避免明显偏袒一边

### D. 整局回放样本

每个样本是整局完整记录中的一个裁量节点。

目标是评估：

- 单个裁量对整局走势的影响
- 某条假信息是否导致游戏过早失衡

---

## 四、说书人的平衡目标

说书人的最高目标应被明确成“可评估目标”，而不是一句抽象口号。

建议拆成 4 项：

### 1. 规则合法性

裁量必须始终在规则允许边界内。

### 2. 公平性

不能通过连续偏向单边的裁量让另一方几乎无反制空间。

### 3. 戏剧性

应尽量避免游戏过早在信息上锁死、或在人数上无悬念地提前结束。

### 4. 可解释性

每次裁量都应能在日志里回答“为什么这次这样做”。

---

## 五、建议的平衡评估指标

为了把“尽兴的游戏”变成可以持续迭代的目标，建议引入一组软指标。

### 局势指标

- `alive_good`
- `alive_evil`
- `confirmed_information_pressure`
- `execution_confidence`
- `evil_exposure_level`

### 对局长度指标

- `ended_before_day_3`
- `ended_before_day_4`
- `reached_final_3`
- `reached_final_4`

### 说书人裁量指标

- `adjudication_count`
- `suppressed_info_count`
- `false_info_count`
- `balance_intervention_count`
- `human_override_count`

### 风险指标

- `hard_lock_detected`
- `single_side_runaway_detected`
- `over_distortion_detected`

这些指标不必一开始就全部用于打分，但至少要进入日志和样本，供后续分析。

---

## 六、实施顺序建议

### Phase 1：样本结构与导出

建立统一样本 schema，并从模拟局或回放中导出裁量时刻包。

交付物：

- `storyteller_eval_samples/` 目录
- schema 文档
- 样本导出脚本

### Phase 2：规则边界校验

给每条样本增加：

- 是否规则合法
- 是否落在允许裁量范围
- 是否使用 legacy fallback

交付物：

- 合法性校验器
- 样本 lint 工具

### Phase 3：平衡性观测

建立一套简单但稳定的“局势热度”与“过早结束风险”指标。

交付物：

- 平衡观测脚本
- 对局摘要报告

### Phase 4：Acceptance Gate

把说书人裁量评估接进 acceptance 流程。

交付物：

- `storyteller_balance_acceptance.py`
- 对应测试

---

## 七、与 W2-A 的直接关系

这套计划不是后续独立研究，而是 `W2-A` 的关键补强。

`W2-A` 当前已完成的是：

- 固定信息与说书人信息路径分离
- judgement ledger 增强
- suppression/distortion 路径显式化

接下来这套模拟数据与评估体系要解决的是：

- 如何验证这些自由裁量“判得好不好”
- 如何避免说书人在合法范围内仍然做出非常糟糕的裁量
- 如何让说书人逐步学会“让一局尽兴地走到后期”

---

## 八、验收标准

这条专项视为初步完成，至少需要满足：

1. 能导出一批完整裁量样本
2. 每条样本都有足够复盘的一局上下文
3. 能对样本做规则边界校验
4. 能对样本做基础平衡性观测
5. 能用 acceptance 脚本自动跑一轮

---

## 结论

说书人智能优化最难的部分，不是“会不会给假信息”，而是：

- 能不能在规则边界内给出合理假信息
- 能不能让善恶双方都持续拥有希望与压力
- 能不能让游戏尽可能走向更有悬念、更有体验的后期

因此，`alpha 0.2` 中的说书人优化，必须建立在**整局上下文样本 + 可复盘裁量日志 + 可观测平衡指标**之上。

这会是后续把说书人从“功能可用”提升到“主持体验可信”的关键基础设施。
