# Alpha 0.2 当前开发进度

## 目的

这份文档用于记录 `alpha 0.2` 当前阶段的真实完成度，作为路线图、任务板和实际实现之间的对齐基线。

它不替代各个 Wave 的任务板，而是回答一个更直接的问题：

- `alpha 0.2` 现在整体做到了哪一步？
- 哪些部分已经可以视为基本完成？
- 哪些部分仍然处于收尾阶段？
- 当前最值得继续推进的重点是什么？

---

## 总体结论

截至当前，`alpha 0.2` 的整体进度可以概括为：

- `Wave 1`：基本完成
- `Wave 2`：基本完成
- `Wave 3`：基本完成
- `Wave 4`：已进入后半段，接近收尾，但尚未封板完成

如果按整体开发完成度估算：

- **计划完成度**：约 `85%+`
- **实际体验完成度**：约 `75% ~ 82%`

这个差值主要来自：

1. 浏览器级真人前端验收尚未最终封板
2. AI 记忆与社交推理虽然已有结构和门禁，但真人体验仍有强化空间
3. 少量角色在实战里的边界问题仍会继续被打出来并需要稳定性修补

也就是说：

- `Wave 3` 的框架、门禁和主链已经完成
- `Wave 3` 的真人体验仍有增强空间

## 当前仍在持续修补的问题类型

虽然四个 Wave 的主结构都已经建立，但当前仍持续出现并需要继续修补的类型包括：

1. **角色技能实战边界**
   - 例如守鸦人、送葬者、占卜师等在真人对局中的事件链、信息链或记忆链偏差

2. **AI 记忆与社交推理**
   - 私密信息保留不足
   - 邪恶队友与 bluff 记忆不够稳
   - 公开跳身份会遗忘

3. **前端真人交互体验**
   - 提名/行动选择器刷新
   - 重试提醒传达
   - 结算 / 历史 / 复盘视图的细节一致性

这些问题更像是：

- `alpha 0.2` 后段的稳定性收尾工作

而不是前 3 个 Wave 的主架构仍然没建好。

## 当前最值得优先继续做的事

按当前状态，后续建议优先顺序如下：

1. **继续收 Wave 4**
   - 历史详情结构
   - 说书人端复盘视图
   - 最终封板验收

2. **继续补实战打出的稳定性问题**
   - 守鸦人 / 送葬者 / 占卜师等角色边界
   - 私密信息显示与使用链
   - 玩家端异常交互

3. **继续增强 Wave 3 的记忆与社交效果**
   - 私密信息长期记忆
   - 邪恶阵营协同记忆
   - 公开跳身份追踪
   - 后续再考虑更重的“记忆落盘 / RL 支撑”
