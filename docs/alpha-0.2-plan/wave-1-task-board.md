# Alpha 0.2 Wave 1 任务板

## Wave 1 目标

确保 `alpha 0.1` 中已经打通的主流程真正稳定，不再出现影响整局可玩性的关键 bug。

Wave 1 聚焦四类问题：

1. 死亡触发链收口
2. 提名 / 投票 / 处决子系统稳定化
3. 前端局间状态污染彻底清理
4. 固定信息与夜晚链路补缺

---

## 当前优先级排序

### P0

1. 守鸦人 / 死亡触发链真人可用性
2. 提名状态与历史状态的彻底分离
3. 新旧对局前端状态隔离

### P1

4. 固定信息角色在真人前端中的显示与时序确认
5. 邪恶首夜信息和间谍魔典刷新稳定性
6. 多轮提名中的 AI / 人类行为一致性

### P2

7. Wave 1 真人自动化验收脚本化

## 执行顺序

建议按下面顺序推进：

1. `W1-A`
2. `W1-C`
3. `W1-B`
4. `W1-D`

原因：

- `W1-A` 和 `W1-C` 都是高优先阻塞项
- `W1-B` 依赖前端状态管理稳定
- `W1-D` 更适合作为 Wave 1 收尾确认项

---

## 完成标准

Wave 1 视为完成，需要同时满足：

1. 真人短局能稳定打完，不再被关键流程 bug 卡住
2. 新旧局状态不会互相污染
3. 提名 / 投票 / 处决链在前端可完整观察
4. 死亡触发和夜晚信息链在真人局中可验证
5. 相关自动化测试与契约测试全部通过
