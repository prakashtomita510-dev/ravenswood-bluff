# Alpha 0.3 开发计划：AI 记忆进化与对局数据工程 (已封板 - 2026-04-24)
 
> [!IMPORTANT]
> **当前状态**：Alpha 0.3 阶段已全面封板并完成验收。
> **发布说明**：详见 [alpha-0.3-release-summary.md](./alpha-0.3-release-summary.md)

## 1. 目标概述
在 Alpha 0.3 阶段，我们将重点攻克 AI 玩家在复杂对局（10-15人）中的信息处理能力，并建立完整的对局数据采集体系，为后续的 AI 强化学习和平衡性分析奠定基础。

## 2. 核心任务清单

### 2.1 动态记忆缩放 (Dynamic Memory Scaling)
- **现状**：AI 的 `WorkingMemory` 和 `SocialGraph` 使用固定的缓存上限，在小局（5-7人）表现良好，但在大局中会导致早期的关键身份申明被挤出上下文。
- **改进方案**：
    - 在 `AIAgent` 初始化时，根据对局人数动态设置 `context_window` 大小。
    - 关键参数公式化：
        - `Recent_Observation_Limit = player_count * 2.5`
        - `Public_Fact_Limit = player_count * 2`
    - 确保 AI 在 Day 3 依然能清晰记得 Day 1 的每个玩家的初始跳法。

### 2.2 全局信息检索 (Global RAG Retrieval)
- **目标**：彻底消除 AI 的“幻觉”和“逻辑断层”。
- **方案**：
    - 引入 **向量化长期记忆 (Vector-based Long-term Memory)**。
    - 将所有 `GameEvent` 和 `ChatMessage` 实时存入内存向量库。
    - AI 在进行决策（`act`）前，通过 RAG 检索与当前讨论焦点（如：特定玩家的身份、特定事件的矛盾点）相关的历史记录。
    - 实现“记忆回溯”功能：AI 可以主动查询“谁在第一天提到了自己是占卜师”。

### 2.3 对局记忆数据库 (Game Intelligence Repository)
- **目标**：将每个 AI 玩家的“一生”记录下来，作为模型训练的宝贵素材。
- **方案**：
    - **数据结构**：
        - `Session_Log`: 包含完整的对局 ID、玩家配置、胜负结果。
        - `Agent_Thought_Trace`: 记录 AI 在每个行动点的 `thought`（思维链）、检索到的记忆片段、以及最终决策。
        - `Social_Graph_Snapshot`: 每轮结束时社交关系的演变快照。
    - **存储实现**：支持导出为 JSONL 格式，并对接本地 PostgreSQL 数据库。
    - **用途**：用于离线分析对局平衡性，并作为强化学习环境的专家经验（Expert Trajectories）。

### 2.4 分层记忆管理与记忆冻结策略 (Layered Memory & Freezing)
- **目标**：在保证“全天候全局记忆”的同时，优化长局中的处理速度，减少冗余计算。
- **方案**：
    - **记忆分层**：
        - **核心层 (Core)**：当前轮次信息、存活玩家动态。
        - **稳定层 (Stable)**：已死亡玩家信息。
    - **记忆冻结 (Memory Freezing)**：
        - 当死亡玩家的身份通过强逻辑（如：送葬者信息、技能闭环）确认为“高置信度”后，AI 将其相关的社交推演暂时冻结，不再在每轮决策中重新分析其早期琐碎发言，仅保留身份摘要。
    - **重燃机制 (Re-thawing)**：
        - 系统建立“逻辑矛盾监听”。一旦出现与“稳定层”信息相悖的强有力新证据，AI 会立即打破冻结，重新对已死亡玩家的身份进行全盘推演。
- **价值**：既保证了 AI 不会“思维僵化”，又显著提升了对局后期的响应效率。

## 3. 剧本与译名优化
- **剧本重命名**：统一更正为 **《暗流涌动》** (Trouble Brewing)。
- **术语对齐**：全面采用中国玩家习惯的术语（如：占卜师、贞洁者、陌客、投毒者等）。

## 4. 进度安排
- **Phase 1 (动态记忆)**：优化现有 `WorkingMemory` 逻辑，实现人数感知。
- **Phase 2 (RAG 引入)**：集成向量引擎，实现全局信息检索。
- **Phase 3 (数据工程)**：上线对局数据采集模块，实现自动化落库。

## 5. 验收标准
1. **全生命周期记忆一致性**：在 15 人满额局中，AI 玩家从开局到**游戏彻底结束（Game Over）的最后一秒**，都必须能准确检索并复述出对局中任何历史阶段的关键事实（如 Day 1 的所有初始跳法、每一轮的投票轨迹等），**记忆置信度不得随天数增加而产生任何衰减**。
2. **零遗忘技术红线**：通过 RAG 动态检索技术，确保所有历史信息在向量库中永久留存。彻底杜绝因为对局时间拉长、信息量过大而导致的“由于内存切片导致的旧信息丢失”现象。
3. **数据完整性溯源**：系统生成的对局报告必须完整包含 AI 在终局时刻调用的所有历史记忆链路，证明其决策是基于全量对局信息的。
